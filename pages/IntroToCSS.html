External, internal & inline

There is three different ways of styling your page. If you'd like to use an external stylesheet you link to it in the html file like this: 

          <!DOCTYPE html>
          <html>
            <head>
              <title> External Stylesheets </title>
              <link rel="stylesheet" type="text/css"
                href="../stylesheets/index.css">
            <head>
            <body>
              <!-- Here is the rest of your code -->
            </body>
          </html>

Your code in your css file, just looks as the first example above.

An internal stylesheet is completely placed within the head section of your html file. It will look something like this: 

          <!DOCTYPE html>
          <html>
            <head>
                <title> External Stylesheets </title>
                <style>
                  p {
                    color: white;
                    width: 80%;
                  }
                  .header {
                    background-color: black;
                  }
                </style>
            <head>
            <body>
                <!-- Here is the rest of your code -->
            </body>
          </html>

You can imagine this might become quite long and your file as your file is doing multiple things at once (both styling and structuring) your app might be quite chaotic!

Inline styling will look like this:

          <!DOCTYPE html>
          <html>
            <head>

What should I use?

To add CSS styles to your website, you can use three different ways to insert the CSS. You can Use an "External Stylesheet", an "Internal Stylesheet", or in "Inline Style". The benefit for using each depends on what you are doing with the Style.The following table explains the difference between them.

External Stylesheet Internal Stylesheet Inline Styles
The External Stylesheet is a .css file that you link your website to. This makes it so that what ever you change in the .css sheet, will effect every page in your website that it is linked to. This prevents you from having to make many code changes in each page. This is for "global" site changes. As you would like your files to do only one thing at the time (for exmaple structuring, but not styling!) this is the most favorable option. This makes it easy to apply styles like classes or id's in order to reuse the code. The downside of using an internal stylesheet is that changes to the internal stylesheet only effect the page the code is inserted into, so your code wouldn't be very DRY, and it isn't clear where styling is happening The Inline style is specific to the tag itself. The inline style uses the HTML "style" attribute to style a specific tag. This is not recommended, as every CSS change has to be made in every tag that has the inline style applied to it. The Inline style is good for one an individual CSS change that you do not use repeatedly through the site.
